Infografía: Sistemas de Nomenclatura y Identificación Única

Garantizando la unicidad global en el intercambio de datos clínicos y administrativos.

Objetivo Principal: Identificación Única Global

El propósito fundamental es especificar cómo se identifican de manera única los participantes y registros en procesos clínicos y administrativos (sistemas, dispositivos, pacientes, profesionales, etc.) para el intercambio de datos. Cada entidad debe tener un identificador ÚNICO en el mundo.

El Concepto Clave: System y Value

Todo identificador se compone de dos partes esenciales para asegurar su unicidad global:

Identifier.System

El sistema responsable de la identificación de la entidad.

Ej: https://www.argentina.gob.ar/interior/renaper (para DNI Argentino)

➡️

Identifier.Value

El identificador único dentro de ese sistema.

Ej: 99999999 M (el número de DNI)

La combinación de `system` y `value` garantiza un identificador **globalmente único**.

Output Esperado: NamingSystem

El resultado de este proceso debe ser un conjunto de `NamingSystem` (Sistema de Nomenclatura), que incluye:

  • URL: La dirección única del sistema de identificación.
  • Descripción: Información sobre el sistema y su propósito.
  • Asociación con los elementos del modelo lógico.

Buenas Prácticas de Nomenclatura de Elementos

Para mantener la uniformidad y claridad en los modelos lógicos:

  • Los elementos que representan fechas deben comenzar con fecha (ej. fechaNacimiento).
  • Los elementos que representan identificadores deben comenzar con id (ej. idPaciente).
  • Los elementos que representan clasificaciones o códigos deben comenzar con cod (ej. codDiagnostico).
